# bundle.py — Сборщик исходного кода в Markdown-бандл

Скрипт `bundle.py` собирает файлы и директории проекта в единый Markdown-документ  
с поддержкой разных кодировок, возможностью точного восстановления оригинала (не реализовано, но файл сохраняется) и гибкой фильтрацией. 
Автоматически нормализует окончания строк, распознаёт бинарные файлы и сохраняет структуру проекта.

## 📦 Установка

```bash
pip install charset-normalizer
```

## 🚀 Быстрый старт

```bash
# Вывести в stdout только .cpp и .h файлы
python bundle.py . -p "*.cpp,*.h"

# Записать всё содержимое src/ в файл (перезапись)
python bundle.py . -p "src/" -o project.md

# Добавить логи к существующему бандлу без base64
python bundle.py . -p "*.log" --no-binary-backup "*.log" -a project.md
```

## 🔧 Основные принципы

### Шаблоны файлов и директорий

- **Шаблон с `/` на конце** → **только директории** (рекурсивно)  
  Пример: `tools/`, `docs/api/`

- **Шаблон без `/`** → **только файлы**  
  Пример: `*.cpp`, `CMakeLists.txt`, `main`

- **Шаблон с `/` внутри (но не на конце) → точный путь к файлу**
  Пример: `src/main.cpp`, `tests/unit/test_core.cpp`

> 💡 Совет: `src/` добавит всю папку `src` со всем содержимым,  
> а `src` добавит только файл с именем `src` в корне (если он существует).

### Порядок обработки

Опции обрабатываются **слева направо**. Каждая опция применяется только к набору путей,  
сформированному **предыдущими** опциями.

Пример:

```bash
python bundle.py . -p "*.txt" --ignore "temp/" -p "legacy/"
```

1. Добавлены все `.txt` файлы в проекте  
2. Из них исключены все, находящиеся в `temp/`  
3. Добавлена вся директория `legacy/` (**игнор из шага 2 на неё не влияет!**)

## ⚙️ Полный список опций

| Опция                | Назначение                                                         |
| -------------------- | ------------------------------------------------------------------ |
| `-p`, `--patterns`   | Добавить файлы/директории по шаблонам (можно несколько раз)        |
| `--ignore`           | Исключить файлы/директории по шаблонам (можно несколько раз)       |
| `--paths-only`       | Добавить только пути без содержимого (по шаблонам)                 |
| `--encoding`         | Задать кодировку: `"шаблон:кодировка"` (например, `"*.bat:cp866"`) |
| `--no-binary-backup` | Отключить base64-дубли для шаблона (сохранять только UTF-8 версию) |
| `-o`, `--output`     | Записать результат в файл (**перезапись**)                         |
| `-a`, `--append`     | Добавить результат к файлу (**дописывание**)                       |
| (без `-o`/`-a`)      | Вывод в **stdout**                                                 |

> ⚠️ `-o` и `-a` взаимоисключающие. Если не указаны — вывод в `stdout`.

## 📚 Примеры использования

### 1. Минимальный бандл только исходников

```bash
python bundle.py . -p "*.cpp,*.h,*.py" -o sources.md
```

→ Собирает все `.cpp`, `.h`, `.py` файлы в проекте, записывает в `sources.md`.

### 2. Полная сборка с исключением временных файлов

```bash
python bundle.py . \
  -p "*" \
  --ignore "__pycache__/,*.tmp,build/,*.log" \
  -o full.md
```

→ Добавляет всё, кроме кэша Python, временных файлов, сборочных директорий и логов.

### 3. Сборка legacy-кода с правильной кодировкой

```bash
python bundle.py . \
  -p "legacy/" \
  --encoding "legacy/:cp866" \
  --encoding "legacy/docs/:windows-1251" \
  -o legacy_bundle.md
```

→ Вся папка `legacy/` интерпретируется как CP866,  
  но подпапка `legacy/docs/` — как Windows-1251.

### 4. Архивация структуры без содержимого

```bash
python bundle.py . \
  -p "src/,config/" \
  --paths-only "src/,config/" \
  -o structure.md
```

→ В бандл попадут только имена файлов и папок, без содержимого.

### 5. Инкрементальная сборка

```bash
# Сначала основной код
python bundle.py . -p "src/" -o project.md

# Потом документация
python bundle.py . -p "docs/" -a project.md

# Потом конфигурация (без base64)
python bundle.py . -p "config/*.ini" --no-binary-backup "config/*.ini" -a project.md
```

→ Результат накапливается в `project.md`.

### 6. Вывод в конвейер (stdout)

```bash
python bundle.py . -p "*.py" | head -n 50
```

→ Полезно для быстрой проверки или передачи другому инструменту.

### 7. Выбор конкретных файлов по полным путям**

```bash
python bundle.py . -p "src/submodule/CMakeLists.txt,src/submodule/tests/CMakeLists.txt" -o bundle.md


## 📂 Формат выходного бандла

Каждый элемент представлен в виде:

```markdown
---
## `путь/к/файлу.cpp`
<!-- bundle:encoding=windows-1251 -->
```cpp
// Код в UTF-8 с LF-окончаниями
```

## `путь/к/файлу.cpp` (original bytes)
```base64
// Оригинальные байты (если не отключено --no-binary-backup)
```
```

Для директорий:

```markdown
---
## `tools/`
```
# directory
```
```

Для `--paths-only`:

```markdown
---
## `debug.log`
```
# path only
```
```

## 🛠 Требования

- Python 3.6+
- Библиотека `charset-normalizer` (`pip install charset-normalizer`)

## ❓ Часто задаваемые вопросы

**Q: Как указать несколько шаблонов?**  
A: Через запятую в одной опции: `-p "*.cpp,tools/"`  
   Или повторяя опцию: `-p "*.cpp" -p "tools/"`

**Q: Почему мой файл не добавился?**  
A: Убедитесь, что:  

- шаблон соответствует типу (файл/директория),  
- путь относителен корня проекта,  
- опция указана **до** `--ignore`, если вы её используете.

**Q: Можно ли использовать `**` для рекурсии?**  
A: Нет, рекурсия включается автоматически через завершающий `/`.  
   `src/` = `src/**/*` в других системах.

**Q: Что делать с файлами в UTF-16?**  
A: Укажите явно: `--encoding="*.bat:utf-16"`

**Q: Зачем нужен base64-блок, если основной текст уже в UTF-8?**  
A: Чтобы можно было **точно восстановить** оригинальный файл (включая кодировку, BOM, CRLF и т.д.). 
   Функция в скрипте не реализована, но файлы будут сохранены и их можно будет восстановить.

**Q: Можно ли отключить base64 полностью?**  
A: Для текстовых файлов — да, через `--no-binary-backup`.  
   Для бинарных файлов — нет, так как они не могут быть представлены как текст.

**Q: Почему мой `.bat`-файл помечен как бинарный?**  
A: Скорее всего, он содержит нулевой байт (`\x00`) или недекодируемые символы.  
   Проверьте кодировку — возможно, он в UTF-16. В этом случае укажите явно: `--encoding="*.bat:utf-16"`.

**Q: Как добавить конкретные файлы по полным путям?**
A: Укажите относительный путь от корня проекта в шаблоне:
`python bundle.py . -p "src/main.cpp,src/utils/helper.cpp"`
Если шаблон содержит `/` (но не заканчивается на `/`), он трактуется как точный путь к файлу, а не как поиск по имени.
