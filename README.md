# bundle.py — Сборщик исходного кода в Markdown-бандл

Утилита собирает файлы и директории проекта в единый Markdown-документ с авто-детектом кодировок, сохранением оригинальных байт в `base64` и гибкой фильтрацией по шаблонам. Кроссплатформенно, нормализует окончания строк, выводит статистику в `stderr`, контент — в `stdout` или файл.

---

## 📦 Установка
```bash
pip install charset-normalizer
```
Требования: Python 3.6+.

---

## 🚀 Быстрый старт
```bash
# Вывести в stdout только C++ заголовки и исходники
python bundle.py . -p "*.cpp,*.h"

# Записать всё содержимое src/ в файл (перезапись)
python bundle.py . -p "src/" -o project.md

# Загрузить шаблоны из файла, отключить base64 для всех
python bundle.py . --patterns-file files.txt --no-binary-backup "*" -o bundle.md
```

## 🔑 Ключевые принципы

### 📐 Правила шаблонов
| Шаблон | Поведение |
|--------|-----------|
| `tools/` | **Директория** (рекурсивно). Совпадает с путями, начинающимися с `tools/` |
| `*.cpp` | **Имя файла**. Ищет по `fnmatch` во всей структуре проекта |
| `src/main.cpp` | **Точный путь**. Совпадает только с файлом по указанному относительному пути |
| `*.*` | **Автоматически нормализуется** в `*` для кроссплатформенной совместимости |

> ⚠️ **Критично:** Опции обрабатываются **строго слева направо**. Каждая следующая применяется только к текущему набору файлов, сформированному предыдущими.

### 🌍 Кодировки
- Автодетект через `charset-normalizer`.
- Файлы `--patterns-file` читаются как `UTF-8` (с BOM или без), с фоллбэком на `windows-1251`.
- Пустые строки и строки, начинающиеся с `#`, игнорируются.
- Окончания строк приводятся к `LF`. Оригинальные байты сохраняются в `base64` (если не отключено).

## ⚙️ Справочник CLI

| Опция | Назначение |
|-------|------------|
| `-p`, `--patterns` | Добавить файлы/директории по шаблонам (через запятую или многократно) |
| `--patterns-file` | Загрузить шаблоны из текстового файла (по одному на строку) |
| `--ignore` | Исключить пути из текущего набора |
| `--paths-only` | Добавить только относительные пути, без содержимого |
| `--encoding` | Принудительно задать кодировку: `"шаблон:кодировка"` |
| `--no-binary-backup` | Отключить `base64`-дубли для текстовых файлов |
| `-o`, `--output` | Записать результат в файл (перезапись) |
| `-a`, `--append` | Дописать результат к существующему файлу |

> 💡 Если `-o` и `-a` не указаны, результат выводится в `stdout`. Опции взаимоисключающие.

## 📖 Примеры использования

### 1. Минимальный бандл исходников
```bash
python bundle.py . -p "*.cpp,*.h,*.py" -o sources.md
```

### 2. Полная сборка с фильтрацией мусора
```bash
python bundle.py . -p "*" --ignore "__pycache__/,*.tmp,build/,*.log" -o full.md
```

### 3. Наследие с явной кодировкой
```bash
python bundle.py . -p "legacy/" --encoding "legacy/:cp866" --encoding "legacy/docs/:windows-1251" -o legacy.md
```

### 4. Инкрементальная сборка
```bash
# Шаг 1: основной код
python bundle.py . -p "src/" -o project.md

# Шаг 2: документация
python bundle.py . -p "docs/" -a project.md

# Шаг 3: конфиги без base64
python bundle.py . -p "config/*.ini" --no-binary-backup "config/*.ini" -a project.md
```

### 5. Только структура проекта
```bash
python bundle.py . -p "src/,include/" --paths-only "src/,include/" -o structure.md
```

## 📄 Формат выходного файла

Каждый файл/директория отделяется `---`. Автоматические мета-комментарии помогают в дальнейшем парсинге:

````markdown
---
## `src/utils/parser.h`
<!-- bundle:encoding=utf-8 -->
```cpp
#pragma once
...
```

---
## `docs/`
```
# directory
```

---
## `archive/legacy.dat`
<!-- bundle:binary=true encoding=unknown -->
```base64
UEsDBBQAAAAIAA...
```
````

- Текстовые файлы не-UTF8 получают блок `base64` с оригинальными байтами.
- Опция `--no-binary-backup` убирает этот дубль.
- Бинарные файлы всегда сохраняются в `base64` (восстановление без оригинальных байт невозможно).

### ❓ Часто задаваемые вопросы

**Q: Нужно ли переходить в папку проекта через `cd`, если указать `.`?**  
**A:** Нет. `.` означает текущую рабочую директорию на момент запуска. Скрипт автоматически преобразует её в абсолютный путь. Можно указать путь явно без `cd`:  
`python bundle.py C:\Projects\MyApp -p "src/" -o out.md`

**Q: Почему `--patterns-file` не находит файлы, хотя они существуют?**  
**A:** Проверьте три момента:
1. **Кодировка файла:** Поддерживается `UTF-8` (с BOM и без) и `Windows-1251`. Скрытые символы или `UTF-16` сломают чтение.
2. **Формат строк:** Пробелы в конце, `\r\n` или пустые строки автоматически обрезаются. Строки, начинающиеся с `#`, считаются комментариями.
3. **Порядок опций:** Файл шаблонов раскрывается в `-p` ровно в том месте, где указан. Если после него идёт `--ignore`, он применится к добавленному набору.

**Q: `--paths-only` без `-p` возвращает ошибку. Почему?**  
**A:** `--paths-only` — это **модификатор вывода**, а не селектор. Он работает только для путей, уже добавленных через `-p` или `--patterns-file`. Без предварительного выбора набор файлов пуст.  
✅ Правильно: `python bundle.py . -p "src/" --paths-only "src/" -o tree.md`

**Q: Почему `*.*` не ловит файлы без расширений?**  
**A:** Python-модуль `fnmatch` трактует `*.*` буквально: «любой символ, точка, любой символ». Файлы вроде `Makefile`, `.gitignore` или `LICENSE` не попадут под маску.  
✅ Используйте `*` для выбора всех файлов, или перечисляйте расширения явно: `*.cpp,*.h,Makefile`.

**Q: Почему шаблоны типа `file*.*` или `*.txt.*` могут не работать как в CMD?**  
**A:** Python использует `fnmatch`, где точка `.` — обычный символ. Шаблон `file*.*` требует наличия точки в имени. Чтобы поведение совпадало с интуицией Windows-пользователей, утилита **автоматически нормализует** маски:  
• `*.*` → `*`  
• `file*.*` → `file*`  
• `test.*` → `test*`  
Это позволяет одному шаблону находить и файлы с расширениями, и без них. Если вам нужна строгая проверка на наличие точки, используйте явные расширения: `*.cpp`, `*.h`.

**Q: `*` найдёт файлы в поддиректориях?**  
**A:** Да, если используется без `/`. Шаблон `*` ищет по имени файла во всей структуре проекта. Для рекурсии по директориям используйте завершающий `/`: `build/`.

**Q: Как скрипт обрабатывает порядок опций?**  
**A:** Строго **слева направо**. Каждая опция изменяет текущий набор `current_set`.  
Пример: `-p "*.txt" --ignore "tmp/" -p "legacy/"`  
1. Добавляет все `.txt`  
2. Удаляет из них находящиеся в `tmp/`  
3. Добавляет всю папку `legacy/` (игнор из шага 2 на неё не влияет)

**Q: Можно ли отключить `base64` для бинарных файлов?**  
**A:** Нет. `--no-binary-backup` отключает дублирование **только для текстовых файлов**, прошедших конвертацию. Настоящие бинарные файлы (изображения, `.exe`, файлы с `\x00`) всегда сохраняются в `base64`, так как без исходных байт их невозможно восстановить.

**Q: Как увидеть, какие файлы реально добавляются?**  
**A:** Групповой список найденных файлов и финальная статистика выводятся в `stderr`. При использовании `-o file.md` консоль не загрязняется Markdown-контентом. Если файл не добавлен, проверьте вывод: может сработать `--ignore`, файл определён как `[BIN]` или `[ERR]`.

**Q: Файл помечен `[BIN]` или `[ERR]`, хотя это текст. Что делать?**  
**A:** `[BIN]` ставится при обнаружении нулевых байтов (`\x00`) или сбое автодетекта. `[ERR]` — при ошибке чтения/декодирования.  
✅ Решение: укажите кодировку явно: `--encoding "legacy.bat:cp866"` или проверьте целостность файла.

**Q: Как добавить конкретные файлы по полным путям?**  
**A:** Укажите относительный путь от корня проекта. Если шаблон содержит `/` (но не заканчивается на ней), он трактуется как точный путь, а не поиск по имени:  
`python bundle.py . -p "src/sub/CMakeLists.txt,src/tests/main.cpp"`

**Q: Поддерживается ли `**` для рекурсии?**  
**A:** Нет. Рекурсия включается автоматически завершающим `/`. `src/` эквивалентно `src/**/*` в других системах. Для файлов используйте `fnmatch`-маски: `**/*.h` не нужен, достаточно `*.h`.

💡 **Совет по отладке:**  
Если хотите проанализировать служебный вывод утилиты, перенаправьте `stderr` в лог:

Так вы отделите служебную статистику от итогового бандла.
